変更履歴とは

ポリシーの読み込み・編集・削除のすべての操作を時系列で記録する監査ログ機能です。

「誰が」「いつ」「なぜ」「何を」変更したかを追跡できます。これにより、ポリシー変更の経緯を透明に管理し、監査やトラブルシューティングに役立てることができます。

一覧表示

「変更履歴」タブを開くと、すべての変更が時系列で一覧表示されます。各エントリには以下の情報が表示されます:

項目 説明
日時 変更が行われた日時
変更者 操作を行ったユーザーの名前
変更理由 変更の目的や背景
ファイル名 インポートされた JSON ファイル名
ポリシー数 影響を受けたポリシーの件数

変更の詳細表示

一覧からエントリをクリックすると、変更の詳細が表示されます。詳細画面では以下が確認できます:

  • 追加されたポリシー: 新しく追加されたポリシーの一覧
  • 削除されたポリシー: 削除されたポリシーの一覧
  • 変更されたポリシー: 変更されたポリシーの一覧（フィールド単位の差分付き）

差分の読み方

変更されたポリシーについては、フィールドごとの差分が表示されます。以下は差分表示の例です:

項目 変更前 変更後
アクセス制御 MFA のみ MFA + 準拠デバイス

変更の種類は以下で分類されます:

  • Added: 新しく追加されたフィールドや値
  • Removed: 削除されたフィールドや値
  • Modified: 変更されたフィールド
  • Unchanged: 変更なし

変更履歴の活用

変更履歴は設計書エクスポート（Excel / PDF）の「変更管理記録」シートに自動的に含まれます。監査対応時にそのまま提出資料として活用できます。

変更履歴を活用することで、以下のような場面で役立ちます:

  • セキュリティ監査への対応: いつ・誰が・なぜ変更したかを証跡として提出
  • トラブルシューティング: 問題が発生した時点の変更内容を特定
  • チーム内の情報共有: ポリシー変更の経緯をチームメンバーと共有

Chapter 7: Change History

CA Policy Analyzer User Manual

What is Change History?

Change History is an audit log feature that records all policy operations — imports, edits, and deletions — in chronological order.

It tracks "who" changed "what," "when," and "why." This enables transparent management of policy change history, useful for auditing and troubleshooting.

List View

Opening the "Change History" tab displays all changes in chronological order. Each entry shows the following information:

Field Description
Date/Time When the change was made
Changed By Name of the user who performed the operation
Reason Purpose or background of the change
File Name Name of the imported JSON file
Policy Count Number of policies affected

Change Details

Click an entry in the list to view the change details. The detail screen shows:

  • Added Policies: List of newly added policies
  • Deleted Policies: List of deleted policies
  • Modified Policies: List of modified policies (with field-level diffs)

Reading the Diff

For modified policies, a field-by-field diff is displayed. Here is an example:

Field Before After
Access Control MFA only MFA + Compliant Device

Changes are classified into the following types:

  • Added: Newly added fields or values
  • Removed: Deleted fields or values
  • Modified: Changed fields
  • Unchanged: No changes

Utilizing Change History

Change history is automatically included in the "Change Management Record" sheet of the design document export (Excel / PDF). It can be used as-is as audit submission material.

Change history is useful in the following scenarios:

  • Security audit response: Submit evidence of who changed what, when, and why
  • Troubleshooting: Identify the changes made at the time a problem occurred
  • Team communication: Share the history of policy changes with team members
